# エックスサーバー デプロイ手順書

GitHubのmainブランチにpushすると、自動でエックスサーバーの本番環境に公開される仕組みを構築する手順書です。

公開URL：`https://クライアントの独自ドメイン/`
公開元：このリポジトリの `public/` フォルダ

---

## 事前準備（クライアント様にご確認いただくこと）

以下の3点をクライアント様にご確認ください。

1. エックスサーバーの **サーバーパネル** にログインできること（Xserverアカウントとは別物）
2. 公開予定の独自ドメインが、すでにメール等で利用中かどうか
3. 独自ドメインのDNS（ネームサーバー）はエックスサーバーに向いていること

> ⚠️ **重要：DNS（ネームサーバー）は絶対に触らないでください。**
> メールが送受信できなくなる重大事故につながります。本作業はFTPでファイルを置くだけなので、DNS変更は一切不要です。

---

## STEP 1 — エックスサーバー側の作業

### 1-0. 国外IPアクセス制限を解除（重要・最初に行う）

GitHub ActionsはアメリカのサーバーからFTP接続しに来ます。エックスサーバーの国外IPブロック設定が有効だと**Actionsが必ず失敗**します。

1. サーバーパネルの「**セキュリティ**」メニュー →「**国外IPアクセス制限設定**」をクリック
2. 対象ドメインを選択
3. 「**FTP**」の制限を **OFF** に変更して保存

### 1-1. サーバーパネルにログイン

エックスサーバーの **サーバーパネル** にログインします。
（契約管理用の「Xserverアカウント」ではなく、サーバー設定用の「サーバーパネル」です）

### 1-2. 独自ドメインの確認

1. 画面右上の「ドメイン」メニュー →「**ドメイン設定**」をクリック
2. 一覧に公開予定の独自ドメインがあるか確認
3. なければ「ドメイン設定追加」タブから追加

### 1-3. サブFTPアカウントの作成（セキュリティ上、必ず作成）

メインFTPアカウントをGitHubに渡すのは危険なため、対象ドメインのフォルダしか操作できない専用アカウントを作ります。

1. 「FTP」メニュー →「**サブFTPアカウント設定**」をクリック
2. 対象の独自ドメインの「選択する」をクリック
3. 「サブFTPアカウント追加」タブを開き、以下を入力

   | 項目 | 値 |
   |---|---|
   | FTPユーザーID | `github`（任意） |
   | パスワード | 任意の強固なパスワード（**メモする**） |
   | 容量 | `0`（無制限のまま） |
   | 接続先ディレクトリ | 変更しない（`/独自ドメイン名/public_html` のまま） |

4. 「確認画面へ進む」→「追加する」で作成

> 実際のユーザー名は `github@独自ドメイン名` の形式になります。

### 1-4. FTPホスト名の確認

1. 「FTP」メニュー →「**FTPソフト設定**」をクリック
2. 対象の独自ドメインを選択
3. 「**FTPサーバー（ホスト名）**」をメモ（例：`sv1234.xserver.jp`）

### 1-5. ✅ エックスサーバー側で取得する情報（チェックリスト）

| # | 項目 | 例 | メモ欄 |
|---|---|---|---|
| ① | FTPホスト名 | `sv1234.xserver.jp` | |
| ② | FTPユーザー名 | `github@client-domain.com` | |
| ③ | FTPパスワード | （1-3で決めたもの） | |

## STEP 2 — GitHub側の作業

### 2-1. リポジトリのSecretsに登録

1. GitHubの対象リポジトリを開く
2. 上部メニュー「**Settings**」をクリック
3. 左メニュー「**Secrets and variables**」→「**Actions**」をクリック
4. 「**New repository secret**」を押し、以下4つを1件ずつ登録

   | Name | Secret（値） |
   |---|---|
   | `FTP_HOST` | STEP 1-5 の ① |
   | `FTP_USER` | STEP 1-5 の ② |
   | `FTP_PASS` | STEP 1-5 の ③ |
   | `FTP_DIR` | `./` |

> `FTP_DIR` を `./` にする理由：サブFTPアカウントは最初から `public_html` 直下に接続されるため、相対パスで「そのまま」を指定すればOKです。

### 2-2. ワークフローファイルの確認（修正不要）

[.github/workflows/deploy.yml](.github/workflows/deploy.yml) はすでに作成済みです。修正は不要です。

```yaml
name: Deploy to Xserver

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Deploy via FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.5
        with:
          server: ${{ secrets.FTP_HOST }}
          username: ${{ secrets.FTP_USER }}
          password: ${{ secrets.FTP_PASS }}
          server-dir: ${{ secrets.FTP_DIR }}
          local-dir: ./public/
          exclude: |
            **/.git*
            **/.git*/**
            **/node_modules/**
```

## STEP 3 — 初回デプロイと動作確認

### 3-1. デプロイ実行

mainブランチに何かしらの変更をpushします。軽微な修正でOKです。

```bash
git commit --allow-empty -m "初回デプロイ実行"
git push origin main
```

### 3-2. Actionsログの確認

1. GitHubリポジトリの「**Actions**」タブを開く
2. 最新の「Deploy to Xserver」ワークフローをクリック
3. 各ステップのステータスを確認
   - ✅ **緑のチェック** → 成功
   - ❌ **赤のバツ** → 失敗（ログを開いて原因確認）

### 3-3. ブラウザで表示確認

`https://クライアントの独自ドメイン/` にアクセスし、サイトが表示されることを確認します。

## トラブルシューティング

| 症状 | 想定原因 | 対処 |
|---|---|---|
| `530 Login authentication failed` | ユーザー名/パスワード誤り | Secretsを再登録。ユーザー名は `github@独自ドメイン` 形式か確認 |
| `ENOTFOUND` / ホストに繋がらない | ホスト名のtypo | `FTP_HOST` の数字部分を再確認 |
| デプロイ成功なのにサイトが表示されない | DNS未設定 / アップロード先違い | サーバーパネルでドメインの公開フォルダを再確認 |
| 古いファイルが残る | キャッシュ | ブラウザのスーパーリロード（Ctrl+Shift+R） |
| 一部ファイルだけ反映されない | `exclude` 設定 | deploy.yml の exclude 句を確認 |

### よくあるミス

- パスワードをコピペしたときに前後に空白が入る
- ユーザー名の `@独自ドメイン` を付け忘れる
- ホスト名の `sv1234` の数字を1桁間違える
- `FTP_DIR` に `/` だけ入れてしまう（正しくは `./`）

---

## 運用フロー（構築完了後）

以降、サイトの更新は以下の流れだけで完了します。

1. ローカルで `public/` 配下のファイルを編集
2. `git add` → `git commit` → `git push origin main`
3. GitHub Actionsが自動でエックスサーバーに反映（数十秒〜1分程度）
4. ブラウザで反映を確認

---

## 緊急時の停止方法

自動デプロイを一時的に止めたい場合：

- GitHubリポジトリの「Settings」→「Actions」→「General」→「Disable actions」
- または、[.github/workflows/deploy.yml](.github/workflows/deploy.yml) の `branches: [main]` を一時的に別名に変更してpush
